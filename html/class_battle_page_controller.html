Detailed Description

flow of one round

both users commit attack the attack each will have a commitment model which will get populated

both users reveal attack the attack each will have a planned attack model which will get populated

one person needs to call run round, if one of them does then the combat state will change to either ended for finished game or back to commit phase

for when the user is stunned is a bit weird, altough i do say we dont want the user to commit and attack, to keep the turn going we still need to commit an attack so for example in the ui we would show something like, You are stunned press here to continue..., but in that action we are actually commiting the attack that has the least cooldown, the attack on the backend will fail but the transaction wont this is the expected behaviour as we dont want to stop the game for the user
